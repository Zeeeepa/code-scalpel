# Documentation Archive

This directory contains historical and temporary documentation from development cycles and release processes. These documents are kept for historical reference but are not part of the active project documentation.

## Directory Structure

### `/v1.1.0/`
Post-mortem and analysis documentation from the v1.1.0 release cycle, including:
- Release execution details
- Issue analysis and investigation summaries
- Pre-release testing reports
- Status tracking documents

**Reference only** - these documents describe completed release activities.

### `/v1.2.0/`
Temporary development documentation created during the v1.2.0 release planning and implementation, including:
- Feature implementation summaries
- Project awareness engine completion details
- CLI unification status reports

**Temporary working documents** - superseded by release notes in `docs/release_notes/RELEASE_NOTES_v1.2.0.md`

### `/internal/`
Internal development documentation that was used during various implementation phases:
- Architecture and implementation notes
- MCP updates and configuration
- Validation and refactoring reports
- Test planning documents

**Working documents** - kept for reference but not actively maintained.

## When to Refer to These Documents

These documents are useful for:
- Understanding historical release decisions and challenges
- Reference during similar future scenarios
- Auditing and compliance purposes
- Learning from past implementation approaches

For current information:
- See `README.md` for project overview
- See `docs/release_notes/` for official release notes
- See `CHANGELOG.md` for version history
- See `docs/` for active project documentation

## Cleanup Policy

Archives are reviewed during major version releases. Documents should be kept for at least 2 major version cycles for historical reference, then considered for removal if no longer relevant.
