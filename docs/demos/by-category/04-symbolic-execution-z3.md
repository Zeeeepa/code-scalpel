## What Is This Demo About?

**Symbolic execution** treats program inputs as mathematical symbols rather than concrete values, then explores every possible execution path simultaneously. At each branch point, it uses the **Z3 SMT (Satisfiability Modulo Theories) solver** — the same solver used by NASA, Microsoft, and Amazon Web Services — to determine:

1. Which inputs reach this branch
2. Whether the branch is even reachable
3. What concrete input values trigger each path

The result: **formal proof** that a function handles all possible inputs correctly (or a concrete counterexample proving it doesn't).

## Technical Depth: How Z3 Solves Constraints

### SMT (Satisfiability Modulo Theories)

Z3 works over mathematical theories:
- **Linear Arithmetic**: `x + 2*y < 10`
- **BitVectors**: integer overflow, bit operations
- **Arrays**: list/dict access and update
- **Strings**: regular expressions, concatenation
- **Uninterpreted Functions**: abstract function calls

### Execution Path Encoding

```python
# Source code
def f(x, y):
    if x > 0:      # branch_1
        if y < x:  # branch_2
            return x - y
        else:
            return y
    return 0
```

Paths encoded as Z3 constraints:
```
Path A: branch_1=True, branch_2=True   → x > 0 AND y < x → Z3: x=5, y=3
Path B: branch_1=True, branch_2=False  → x > 0 AND y >= x → Z3: x=3, y=5
Path C: branch_1=False                 → x <= 0 → Z3: x=-1, y=anything
```

### Path Explosion Problem

For n branches, there are up to 2^n paths. Mitigations:
- **Loop unrolling**: fixed bound (default: 3 iterations)
- **Path merging**: collapse equivalent paths
- **Lazy expansion**: only expand relevant branches
- **Max depth limit**: configurable (default: 50 hops)
