Quick Start Guide

Get Code Scalpel running in under 5 minutes. This guide covers installation, basic configuration, and your first MCP tool call.

Installation

Code Scalpel is distributed as a single MIT-licensed Python package. Install it from PyPI:

bash 
# Install from PyPI
pip install code-scalpel

# Or with uvx for isolated execution
uvx code-scalpel mcp
üí° Tip

Using uvx is recommended for MCP server usage as it handles dependency isolation automatically.

Start the MCP Server

Code Scalpel runs as an MCP server that AI agents connect to. Start it with:

bash 
# Start with stdio transport (default)
code-scalpel mcp

# Start with HTTP transport
code-scalpel mcp --transport streamable-http --port 8593

# Set tier (community, pro, enterprise)
export CODE_SCALPEL_TIER=pro
code-scalpel mcp

2

Example: Security Scan

python 
# Vulnerable code example
user_id = request.args.get('id')
query = f"SELECT * FROM users WHERE id={user_id}"
cursor.execute(query)

# Code Scalpel detects:
# [CRITICAL] SQL Injection (CWE-89) at line 3
#   Taint Path: request.args ‚Üí user_id ‚Üí query ‚Üí cursor.execute()
#   Recommendation: Use parameterized queries
‚úì What's Different

Traditional tools use regex patterns and miss this vulnerability because it spans 3 lines. Code Scalpel tracks the taint through variable assignments.

Understanding Tiers

Code Scalpel uses an open-core model. All code ships together, but features are restricted at runtime based on your configured tier.

bash 
# Community (default) - 10 tools, single-file analysis
code-scalpel mcp

# Pro - 17 tools, project-wide analysis, symbolic execution
export CODE_SCALPEL_TIER=pro
code-scalpel mcp

# Enterprise - 20 tools, cross-file security, policy governance
export CODE_SCALPEL_TIER=enterprise
code-scalpel mcp
