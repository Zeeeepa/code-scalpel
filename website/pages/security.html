Governance for AI code operations

AI agents are becoming operators — code-changing actors, not just suggestion engines. That's powerful, but it changes the risk profile. Code Scalpel is built from the ground up to make agent operations safe, bounded, and auditable.

Core principles

  • No execution by default: analysis tools parse code; they don’t run it.
  • Bounded writes: edits target one symbol (function/class/method) instead of whole-file rewrites.
  • Pre-flight checks: safety gates can validate changes before writing.
  • Tier transparency: limits and behavior are explicit by tier.

Enterprise governance

  • Fail-closed policy integrity: cryptographic verification detects tampered policy files.
  • Compliance modes: code_policy_check maps findings to HIPAA, SOC2, GDPR, PCI-DSS.
  • Audit artifacts: generate evidence for reviews and compliance reports.

The thesis: you can't govern operators with suggestions

Traditional security advice (prompt the model to be careful, review outputs, test after deployment) doesn't scale when agents operate autonomously. You need structural controls — limits enforced by the server, not requested from the model.
